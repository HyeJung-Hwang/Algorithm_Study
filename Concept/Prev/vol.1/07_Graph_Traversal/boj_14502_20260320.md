# BOJ 14502 - 연구소 (2026-03-20)

## 핵심 아이디어: 브루트포스 + 멀티소스 BFS

### 풀이 흐름
```
1. 빈 칸(0) 목록 추출
2. 빈 칸 중 3개 조합 선택 → 벽으로 설치
3. 바이러스(2) 전부 큐에 넣고 멀티소스 BFS → 퍼질 수 있는 0을 전부 2로 감염
4. 남은 0 개수 = 안전영역
5. 모든 조합 중 최댓값 출력
```

---

## 브루트포스 + BFS vs 상태 확장 BFS

| | 브루트포스 + BFS | 상태 확장 BFS |
|---|---|---|
| 경우의 수 | 작음 (수만 이하) | 큼 |
| 구현 | 단순 | 복잡 |
| 예시 | 14502 (N,M ≤ 8) | 2206 (N,M ≤ 1000) |

**판단 기준: 범위가 작으면 브루트포스, 크면 상태 확장**

- 14502: 빈 칸 최대 64개 중 3개 선택 → `C(64,3) = 41,664`가지 → 충분히 빠름
- 2206: 벽이 최대 N*M개 → 미리 고르면 너무 느림 → 상태에 flag 포함

---

## 멀티소스 BFS vs 싱글소스 BFS

**결과는 같지만, 멀티소스가 더 효율적.**

```python
# 멀티소스: 바이러스 전부 한 번에 큐에 넣고 BFS 한 번
queue = deque()
for r in range(N):
    for c in range(M):
        if graph[r][c] == 2:
            queue.append((r, c))
bfs(queue)  # 한 번만 실행

# 싱글소스: 바이러스마다 BFS 따로
for r in range(N):
    for c in range(M):
        if graph[r][c] == 2:
            bfs(deque([(r, c)]))  # 바이러스 수만큼 실행
```

**14502에서 차이 없는 이유:** 바이러스는 갈 수 있는 모든 빈 칸을 다 감염시키기 때문에 출발점이 몇 개든 최종 결과 동일.

**거리가 중요한 문제였다면 달라짐:**
```
[A=2]  0  0  0  [B=2]

멀티소스 dist: 1  2  2  1   ← 정확한 최단거리 (각 칸의 가장 가까운 바이러스 기준)
A 먼저 싱글소스: 1  2  3  4  → B 덮어쓰면 맞긴 하지만 구현 복잡
```

---

## dist 배열이 필요한가?

**14502는 dist 불필요.** "감염됐냐 안 됐냐"만 중요.

- dist가 필요한 경우: 최단거리를 저장해야 할 때 (ex. 1697번 숨바꼭질, 2206번)
- 14502는 안전영역(0의 개수) 카운트만 하면 됨

## combinations 문법

```python
from itertools import combinations

lst = [1, 2, 3, 4]
for combo in combinations(lst, 2):
    print(combo)
# (1, 2) / (1, 3) / (1, 4) / (2, 3) / (2, 4) / (3, 4)
```

- `combinations(iterable, r)` → r개 뽑는 순서 없는 조합
- 결과는 튜플로 나옴

```python
zeros = [(r, c) for r in range(N) for c in range(M) if graph[r][c] == 0]

for walls in combinations(zeros, 3):
    for r, c in walls:  # walls = ((r1,c1), (r2,c2), (r3,c3))
        tmp[r][c] = 1
```

## 딕셔너리 key로 튜플 가능한가?

**가능.** hashable 하면 key가 될 수 있음.

| 가능 | 불가능 |
|---|---|
| `tuple`, `int`, `str` (불변) | `list`, `dict`, `set` (가변) |

```python
d = {}
d[(1, 2)] = "hello"   # OK
d[[1, 2]] = "hello"   # TypeError: unhashable type: 'list'
d[([1,2], 3)] = "hi"  # TypeError: 튜플 안에 리스트 있으면 불가
```

## 실수 모음

| 실수 | 원인 |
|---|---|
| `nx, ny = x + dx[_], dy[_]` | `y +` 빠짐 → ny가 항상 dy값만 됨 |
| `visited`를 main에서 한 번만 초기화 | 조합마다 초기화해야 함. 안 하면 이전 BFS 방문 정보가 남음 |
| `graph[m][n]` | n이 행, m이 열인데 인덱스 뒤집힘 → `graph[n][m]` |
| `graph`를 복사 없이 BFS에 넘김 | 원본이 오염됨 → `tmp = [row[:] for row in graph]` 로 복사 후 넘기기 |

---

## 최종 코드

```python
import sys
input = sys.stdin.readline
from collections import deque
from itertools import combinations

dx = [-1,1,0,0]
dy = [0,0,-1,1]

def bfs(input_graph):
    queue = deque()
    for i in range(N):
        for j in range(M):
            if input_graph[i][j] == 2:
                queue.append((i, j))

    while queue:
        x, y = queue.popleft()
        for d in range(4):
            nx, ny = x + dx[d], y + dy[d]
            if 0 <= nx < N and 0 <= ny < M:
                if input_graph[nx][ny] == 0:
                    input_graph[nx][ny] = 2
                    queue.append((nx, ny))

    return sum(row.count(0) for row in input_graph)

if __name__ == "__main__":
    N, M = map(int, input().split())
    graph = [list(map(int, input().split())) for _ in range(N)]

    walls = combinations([(n, m) for n in range(N) for m in range(M) if graph[n][m] == 0], 3)

    ans = 0
    for wall_case in walls:
        tmp = [row[:] for row in graph]
        for i, j in wall_case:
            tmp[i][j] = 1
        ans = max(ans, bfs(tmp))

    print(ans)
```

> visited 배열 따로 필요 없음: `input_graph[nx][ny] == 0` 조건이 미방문 체크 역할을 함 (0 → 2로 바꾸므로 재방문 불가)
